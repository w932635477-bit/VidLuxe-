# Skills 对比分析：核心 Skills vs Using-Superpowers

> 本文档深入对比三个核心工作流 Skills 与 Using-Superpowers 元技能的区别、关系和协作方式。

---

## 1. 核心概念对比

### 定义差异

| Skill 类型 | 定义 | 作用 |
|-----------|------|------|
| **Using-Superpowers** | 元技能（Meta-Skill） | 决定**何时**、**如何**使用其他 Skills |
| **Autonomous TDD** | 工作流技能 | 自动执行**测试驱动开发** |
| **Parallel Refactor** | 工作流技能 | 自动执行**大规模重构** |
| **Deploy Checklist** | 检查技能 | 标准化**部署流程** |

## 3. 详细的对比分析

### Using-Superpowers（元技能）

#### 核心职责

```markdown
1. 判断职责
   - 评估当前任务是否有 Skill 适用
   - 即使 1% 可能性也要检查
   - 确定优先级（流程 Skill > 实现 Skill）

2. 强制原则
   - "IF A SKILL APPLIES, YOU DO NOT HAVE A CHOICE"
   - 不可跳过 Skill 检查
   - Skill 检查在澄清问题之前

3. 风险警示
   - 识别"逃避使用 Skill"的思维
   - 防止"这太简单了"的误判
   - 阻止未经验证的行动
```

#### 判断流程

```
收到用户消息
     │
     ▼
┌─────────────────┐
│ 有任何 Skill    │──── 否 ────▶ 直接响应
│ 可能适用吗？    │
└────────┬────────┘
         │ 是
         ▼
┌─────────────────┐
│ 调用 Skill Tool │
│ 加载 Skill 内容 │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 宣布使用 Skill  │
│ "Using [skill]" │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 按流程执行任务  │
└─────────────────┘
```

### 核心 Skills（工作流技能）

#### 职责对比

| Skill | 核心职责 | 执行方式 |
|-------|---------|----------|
| **Autonomous TDD** | 自动实现功能 | 测试→实现→修复→循环 |
| **Parallel Refactor** | 并行重构代码 | 规划→分配→并行→合并 |
| **Deploy Checklist** | 标准化部署 | 检查→部署→验证 |

#### 执行特点

```markdown
Autonomous TDD:
- 完全自动化循环
- 测试驱动实现
- 失败自动换方案
- 无需人工干预

Parallel Refactor:
- 多 Agent 并行
- 依赖关系协调
- 进度可视化
- 需要前期规划

Deploy Checklist:
- 标准化步骤
- 每步确认
- 问题早发现
- 可回滚
```

## 4. 协作关系

### Using-Superpowers 如何调度核心 Skills

```
用户请求："帮我实现一个登录功能"
           │
           ▼
┌─────────────────────────────────────┐
│        Using-Superpowers            │
│                                     │
│  分析：这是一个"新功能开发"任务      │
│  匹配：autonomous-tdd 可能适用       │
│  决策：调用 autonomous-tdd          │
│                                     │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│          Autonomous TDD             │
│                                     │
│  执行：                             │
│  1. 编写测试用例                    │
│  2. 实现功能代码                    │
│  3. 运行测试                        │
│  4. 失败则修复并重试                │
│  5. 循环直到所有测试通过            │
│                                     │
└─────────────────────────────────────┘
```

### 典型协作场景

#### 场景 1：新功能开发

```
用户："实现图片批量上传功能"
         │
         ▼
Using-Superpowers 判断：
  ✓ 这是新功能开发
  ✓ autonomous-tdd 适用
  ✓ 优先级：高
         │
         ▼
调用 /autonomous-tdd
         │
         ▼
Autonomous TDD 执行：
  1. 编写上传测试
  2. 实现上传逻辑
  3. 测试通过
```

#### 场景 2：大规模重构

```
用户："把 Redux 迁移到 Zustand"
         │
         ▼
Using-Superpowers 判断：
  ✓ 这是大规模代码变更
  ✓ parallel-refactor 适用
  ✓ 涉及文件 >10 个
         │
         ▼
调用 /parallel-refactor
         │
         ▼
Parallel Refactor 执行：
  1. 分析依赖关系
  2. 分配 4 个 Agent
  3. 并行重构
  4. 合并结果
```

#### 场景 3：部署

```
用户："帮我部署到生产环境"
         │
         ▼
Using-Superpowers 判断：
  ✓ 这是部署任务
  ✓ deploy-checklist 适用
  ✓ 安全优先
         │
         ▼
调用 /deploy-checklist
         │
         ▼
Deploy Checklist 执行：
  1. 检查构建
  2. 检查服务器
  3. 执行部署
  4. 验证结果
```

## 5. 使用场景对比

### 何时依靠 Using-Superpowers（自动判断）

| 场景 | 说明 |
|------|------|
| **不确定用什么 Skill** | 让系统判断 |
| **任务模糊** | 先检查再行动 |
| **新手使用** | 避免遗漏 Skill |
| **复杂任务** | 系统分析后推荐 |

### 何时直接调用核心 Skills（显式调用）

| 场景 | 说明 |
|------|------|
| **明确知道要做什么** | 直接执行更高效 |
| **需要自定义参数** | 显式调用可传参 |
| **重复相同任务** | 形成固定工作流 |
| **高级用户** | 跳过判断，直接行动 |

## 6. 优劣势对比

### Using-Superpowers

| 优势 | 劣势 |
|------|------|
| ✅ 自动判断，无需思考 | ❌ 判断可能不准确 |
| ✅ 强制 Skill 使用，避免遗漏 | ❌ 增加检查开销 |
| ✅ 适合新手 | ❌ 高级用户可能觉得多余 |
| ✅ 防止"偷懒"跳过流程 | ❌ 简单任务也可能触发 |

### Autonomous TDD

| 优势 | 劣势 |
|------|------|
| ✅ 完全自动执行 | ❌ 需要详细需求描述 |
| ✅ 测试保证质量 | ❌ 可能走偏方向 |
| ✅ 自我修正 | ❌ 复杂业务不适用 |
| ✅ 可追溯过程 | ❌ 迭代耗时 |

### Parallel Refactor

| 优势 | 劣势 |
|------|------|
| ✅ 并行加速 4x | ❌ 仅适合大规模任务 |
| ✅ 自动协调依赖 | ❌ 规划复杂 |
| ✅ 进度可视 | ❌ 回滚成本高 |
| ✅ 适合大型迁移 | ❌ 需要测试覆盖 |

### Deploy Checklist

| 优势 | 劣势 |
|------|------|
| ✅ 标准化流程 | ❌ 步骤固定 |
| ✅ 问题早发现 | ❌ 检查耗时 |
| ✅ 可回滚 | ❌ 部分需手动 |
| ✅ 文档化 | ❌ 不够灵活 |

## 8. 总结

### 核心区别

| 对比项 | Using-Superpowers | 核心 Skills |
|--------|------------------|-------------|
| **本质** | 决策框架 | 执行工具 |
| **触发** | 自动判断 | 显式调用 |
| **作用** | 决定用什么 | 执行具体任务 |
| **强制性** | 强制检查 | 强制执行流程 |

### 协作本质

```
Using-Superpowers = 调度员
    │
    ├──▶ Autonomous TDD = 开发工程师（自动化）
    ├──▶ Parallel Refactor = 架构师（并行化）
    └──▶ Deploy Checklist = 运维工程师（标准化）
```

### 使用建议

| 用户类型 | 建议 |
|---------|------|
| **新手** | 依靠 Using-Superpowers 自动判断 |
| **中级** | 了解各 Skill，需要时显式调用 |
| **高级** | 形成固定工作流，直接调用 Skill |
