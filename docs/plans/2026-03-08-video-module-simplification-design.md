# 视频模块简化设计方案

> **文档版本**: v1.0
> **创建日期**: 2026-03-08
> **作者**: Claude + 用户讨论

## 概述

移除视频渲染和调色功能，专注于关键帧提取和图片增强，提供用户可直接下载的增强图片素材。

## 背景与问题

### 当前问题
- 视频渲染耗时太长（2-5 分钟）
- FFmpeg 视频合成容易出错
- 用户体验不佳（等待时间长，失败率高）

### 解决方案
- 移除视频调色、帧替换合成、封面嵌入视频功能
- 专注于图片增强输出
- 让用户使用自己的剪辑软件处理增强后的图片

## 功能变更

### 移除的功能
- `/api/video/color-grade` - 视频调色
- `/api/video/replace-frames` - 帧替换合成视频
- `/api/video/embed-cover` - 封面嵌入视频

### 保留的功能
- `/api/video/analyze` - 关键帧提取
- `/api/video/enhance-cover` - 图片增强

### 新增的功能
- 批量下载增强图片（逐张下载，不打包）

## 新流程设计

```
上传视频 → 提取关键帧 → 选择封面/替换帧 → 批量增强图片 → 滑动查看 → 一键下载
                                                                ↓
                                              显示额度扣除，余额提示
```

## UI 设计

### 结果页布局
```
┌─────────────────────────────────────────────────────────────┐
│                      滑动卡片区域                            │
│  ┌─────────────────────────────────────────────────────┐  │
│    [ 增强#1 ] [ 增强#2 ] [ 增强#3 ] [ ... ]   ← 滑动 →  │
│                  显示当前：封面/替换帧 N                         │
└─────────────────────────────────────────────────────────────┘
│              "一键下载全部图片" 按钮                             │
│            (消耗 X 额度，当前余额：X)                       │
└─────────────────────────────────────────────────────────────┘
```

### 交互流程
1. **滑动查看** - 左右滑动浏览所有增强图片
2. **点击下载** - 弹出确认对话框
3. **确认下载** - 显示扣除额度，逐张下载图片
4. **额度不足** - 弹窗提示，跳转付费页面
5. **额度扣除** - 下载页正常显示用户所剩余额度，同步扣除和显示

## 计费规则

| 规则 | 说明 |
|------|------|
| **按张计费** | 每张增强图消耗 1 额度 |
| 总消耗 | 选中的封面帧 + 替换帧数量 |

## 技术实现要点

### 后端修改
1. 修改 `enhance-cover` API 支持批量处理
2. 移除视频合成相关代码
3. 实现图片逐张下载接口

### 前端修改
1. 修改 `VideoFlow` 组件，移除视频合成逻辑
2. 重新设计 `ResultSection`，支持滑动卡片 + 一键下载
3. 实现下载确认弹窗和额度显示

### 状态管理
1. 简化 `video-store`，移除视频合成相关状态
2. 新增批量下载状态管理

## 预期效果

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 处理时间 | 2-5 分钟 | < 70 秒 |
| 错误率 | 高（FFmpeg 复杂命令） | 低（仅图片增强） |
| 用户控制 | 有限 | 完全控制素材 |
