# 腾讯云 CDN 缓存规则配置 - 操作指南

## 📋 配置前准备

1. 登录腾讯云控制台：https://console.cloud.tencent.com/cdn
2. 找到域名：`vidluxe.com.cn`
3. 点击「管理」进入配置页面

---

## 🎯 第一步：清除现有缓存规则

1. 进入「缓存配置」→「缓存规则配置」
2. 删除所有现有规则（如果有）
3. 准备添加新规则

---

## 🔧 第二步：添加缓存规则（按顺序）

### 规则 1：Next.js 静态资源 - 长期缓存 ⭐

**目的**：CSS、JS 文件包含哈希值，可以安全地长期缓存

**配置**：
```
类型：目录
内容：/_next/static/
缓存过期时间：自定义
  - 缓存时间：365 天
  - 强制缓存：开启
浏览器缓存：遵循源站
优先级：1（最高）
```

**操作步骤**：
1. 点击「新增规则」
2. 类型选择「目录」
3. 内容填写：`/_next/static/`
4. 缓存过期时间选择「自定义」
5. 输入：`31536000` 秒（365 天）
6. 点击「保存」

---

### 规则 2：用户上传文件 - 短期缓存

**目的**：用户上传的图片、视频缓存 1 天

**配置**：
```
类型：目录
内容：/uploads/
缓存过期时间：自定义
  - 缓存时间：1 天
浏览器缓存：遵循源站
优先级：2
```

**操作步骤**：
1. 点击「新增规则」
2. 类型选择「目录」
3. 内容填写：`/uploads/`
4. 缓存过期时间选择「自定义」
5. 输入：`86400` 秒（1 天）
6. 点击「保存」

---

### 规则 3：API 请求 - 不缓存 ⭐⭐⭐

**目的**：API 返回动态数据，绝对不能缓存

**配置**：
```
类型：目录
内容：/api/
缓存过期时间：不缓存
浏览器缓存：不缓存
优先级：3
```

**操作步骤**：
1. 点击「新增规则」
2. 类型选择「目录」
3. 内容填写：`/api/`
4. 缓存过期时间选择「不缓存」
5. 点击「保存」

---

### 规则 4：HTML 页面 - 不缓存 ⭐⭐⭐

**目的**：HTML 包含对静态资源的引用，不能缓存

**配置**：
```
类型：文件后缀
内容：html
缓存过期时间：不缓存
浏览器缓存：不缓存
优先级：4
```

**操作步骤**：
1. 点击「新增规则」
2. 类型选择「文件后缀」
3. 内容填写：`html`
4. 缓存过期时间选择「不缓存」
5. 点击「保存」

---

### 规则 5：首页和关键页面 - 不缓存

**目的**：确保首页和关键页面不被缓存

**需要添加的路径**：
- `/` （首页）
- `/auth` （登录注册）
- `/try` （功能页面）
- `/pricing` （定价页面）
- `/dashboard` （用户中心）

**配置（每个路径单独添加）**：
```
类型：文件
内容：/
缓存过期时间：不缓存
浏览器缓存：不缓存
优先级：5
```

**操作步骤**（重复 5 次）：
1. 点击「新增规则」
2. 类型选择「文件」
3. 内容填写：`/`（然后是 `/auth`、`/try`、`/pricing`、`/dashboard`）
4. 缓存过期时间选择「不缓存」
5. 点击「保存」

---

### 规则 6：其他资源 - 默认缓存

**目的**：其他未匹配的资源使用短期缓存

**配置**：
```
类型：全部文件
内容：*
缓存过期时间：自定义
  - 缓存时间：10 分钟
浏览器缓存：遵循源站
优先级：99（最低）
```

**操作步骤**：
1. 点击「新增规则」
2. 类型选择「全部文件」
3. 缓存过期时间选择「自定义」
4. 输入：`600` 秒（10 分钟）
5. 点击「保存」

---

## 🔄 第三步：刷新 CDN 缓存

配置完成后，必须刷新缓存才能生效：

### 方法 1：目录刷新（推荐）

1. 进入「缓存刷新」页面
2. 选择「目录刷新」
3. 输入以下 URL（每行一个）：
   ```
   https://vidluxe.com.cn/
   https://vidluxe.com.cn/_next/static/
   ```
4. 点击「提交」

### 方法 2：URL 刷新（备选）

如果目录刷新不够，可以刷新具体 URL：
```
https://vidluxe.com.cn/
https://vidluxe.com.cn/auth
https://vidluxe.com.cn/try
https://vidluxe.com.cn/pricing
```

## ✅ 第四步：验证配置

等待 5-10 分钟后，验证配置是否生效：

### 1. 验证静态资源缓存

```bash
curl -I https://vidluxe.com.cn/_next/static/css/app-layout.css
```

**预期响应头**：
```
HTTP/1.1 200 OK
Cache-Control: public, max-age=31536000, immutable
X-Cache-Lookup: Cache Hit  # 第二次请求时
Age: xxx  # 缓存时间
```

### 2. 验证 HTML 不缓存

```bash
curl -I https://vidluxe.com.cn/
```

**预期响应头**：
```
HTTP/1.1 200 OK
Cache-Control: no-cache, no-store, must-revalidate
X-Cache-Lookup: Cache Miss  # 每次都应该是 Miss
```

### 3. 验证 API 不缓存

```bash
curl -I https://vidluxe.com.cn/api/health
```

**预期响应头**：
```
HTTP/1.1 200 OK
Cache-Control: no-cache
X-Cache-Lookup: Cache Miss
```

### 4. 浏览器测试

1. 打开浏览器开发者工具（F12）
2. 切换到 Network 标签
3. 访问 https://vidluxe.com.cn/
4. 刷新页面（Cmd/Ctrl + R）
5. 检查：
   - `_next/static/` 的资源应该显示「from disk cache」或「304 Not Modified」
   - HTML 文件应该是「200 OK」且每次都重新加载

## 🎨 高级配置（可选）

### 1. 智能压缩

**位置**：高级配置 → 智能压缩

**配置**：
- Gzip 压缩：开启
- Brotli 压缩：开启（如果支持）
- 压缩类型：text/html, text/css, application/javascript, application/json

### 2. Range 回源

**位置**：回源配置 → Range 回源

**配置**：
- Range 回源：开启
- 说明：支持大文件断点续传

### 3. 视频拖拽

**位置**：高级配置 → 视频拖拽

**配置**：
- 视频拖拽：开启
- 说明：支持视频快进

### 4. HTTPS 配置

**位置**：HTTPS 配置

**配置**：
- 强制 HTTPS：开启
- HTTP 2.0：开启
- HSTS：开启
  - max-age：31536000
  - includeSubDomains：开启

### 5. 访问控制（可选）

**位置**：访问控制 → IP 访问限频

**配置**：
- 单 IP 访问频率：100 次/秒
- 超过限制返回：429

## 📈 监控和优化

### 查看缓存命中率

1. 进入「数据分析」→「访问情况分析」
2. 查看「缓存命中率」
3. **目标**：> 90%

### 查看回源带宽

1. 进入「数据分析」→「回源分析」
2. 查看「回源带宽」
3. **目标**：< 10% 总带宽

### 优化建议

如果缓存命中率低于 90%：

1. 检查是否有大量动态请求
2. 增加静态资源的缓存时间
3. 检查是否有查询参数导致缓存失效

## ⚠️ 常见问题

### Q1: 配置后页面还是显示旧内容

**原因**：浏览器缓存或 CDN 缓存未刷新

**解决**：
1. 清除浏览器缓存（Cmd/Ctrl + Shift + R）
2. 刷新 CDN 缓存（目录刷新）
3. 等待 5-10 分钟

### Q2: 静态资源还是 404

**原因**：HTML 被缓存，引用了旧的静态资源

**解决**：
1. 确认 HTML 缓存规则为「不缓存」
2. 刷新 CDN 缓存
3. 重新部署应用

### Q3: API 响应被缓存

**原因**：API 缓存规则未生效

**解决**：
1. 检查 `/api/` 规则是否存在
2. 确认优先级高于默认规则
3. 刷新 CDN 缓存
