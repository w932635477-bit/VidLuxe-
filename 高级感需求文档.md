## 一、项目定位

### 1.1 核心问题

**当前市场痛点：**

市面上已有大量 AI 剪辑工具，但它们共同面临一个问题：**做出来的视频"能用"但"不够高级"**。

- 模板千篇一律，缺乏差异化
- AI 理解力有限，只能做机械式剪辑
- 用户期望是"爆款"，AI 只能交付"及格线"

**用户真正的需求：**

```
"我做出来的视频，不要看起来像 AI 做的"
"我想要那种...说不上来但一看就很专业的感觉"
```

这个"说不上来的感觉" = **高级感**

### 1.2 产品重新定位

```
之前：AI Director - 让 AI 像导演一样剪辑
      （目标太宏大，技术差距大）

现在：高级感引擎 - 让任何内容瞬间变得高级
      （目标具体，AI 可学习，用户可感知）
```

**核心价值主张：**

```
上传你的普通视频/PPT/设计稿
    ↓
AI 分析并应用"高级感规则"
    ↓
输出一个"看起来很贵"的版本
```

### 1.3 差异化优势

| 维度 | 传统 AI 剪辑 | VidSlide 高级感引擎 |
|------|-------------|-------------------|
| 目标 | 自动化剪辑 | 提升品质感 |
| 用户期望 | 帮我剪出来 | 让我看起来更专业 |
| AI 能力 | 机械执行 | 学习审美规则 |
| 可感知性 | 功能导向 | 结果导向 |
| 差异化 | 低（同质化严重） | 高（聚焦高级感） |

## 三、高级感定义

### 3.1 什么是高级感

**高级感不是玄学，是可量化的设计规则。**

```
高级感 = 克制 + 精致 + 一致性

克制：
  - 颜色少一点（不超过 3 种主色）
  - 元素少一点（留白的艺术）
  - 动画慢一点（不急躁）

精致：
  - 边缘干净（抠图精细）
  - 间距舒适（呼吸感）
  - 阴影柔和（有层次）

一致性：
  - 色温统一（不忽冷忽热）
  - 风格统一（不杂乱）
  - 节奏统一（不突兀）
```

### 3.2 高级感 vs 低级感对比

| 维度 | 低级感 | 高级感 |
|------|--------|--------|
| **色彩** | 高饱和度、颜色多、色温不统一 | 低饱和度、颜色克制、色温统一 |
| **排版** | 系统字体、间距紧、高亮刺眼 | 精致字体、有呼吸感、低饱和强调 |
| **构图** | 随意、背景杂乱、焦点不明确 | 三分法、背景简洁、焦点清晰 |
| **动效** | 线性动画、时长不当、转场生硬 | 缓动曲线、恰到好处、转场柔和 |
| **音频** | 有噪音、音量不稳、BGM 不搭 | 人声清晰、音量稳定、情绪匹配 |
| **细节** | 边缘粗糙、阴影不自然 | 边缘干净、细节精致 |

### 4.2 色彩高级感实现

#### 4.2.1 高级感色彩规则

```yaml
# 高级感色彩配置文件
premium_color_profiles:

  minimal:
    # 极简风格（Apple 风格）
    saturation: 0.65        # 饱和度降至 65%
    contrast: 0.90          # 适度对比度
    temperature: 5200       # 中性偏冷色温
    highlights: -10         # 压低高光
    shadows: 5              # 提亮阴影
    color_count_max: 3      # 主色不超过 3 种

  warm_luxury:
    # 暖调奢华风格
    saturation: 0.70
    contrast: 0.85
    temperature: 5800       # 暖色调
    highlights: -5
    shadows: 10
    color_count_max: 3

  cool_professional:
    # 冷调专业风格
    saturation: 0.60
    contrast: 0.95
    temperature: 4800       # 冷色调
    highlights: -15
    shadows: 0
    color_count_max: 4

  morandi:
    # 莫兰迪色系
    saturation: 0.45        # 低饱和度
    contrast: 0.80
    temperature: 5000
    gray_tint: 0.15         # 加入灰调
    color_count_max: 3
```

### 5.4 规则学习

#### 5.4.1 对比学习方法

```python
class PremiumRuleLearner:
    """高级感规则学习器"""

    def learn_from_comparison(self, premium_videos, normal_videos):
        """通过对比学习"""

        # 提取特征
        premium_features = [self.extract_features(v) for v in premium_videos]
        normal_features = [self.extract_features(v) for v in normal_videos]

        rules = {}

        # 对每个特征进行统计检验
        for feature_name in premium_features[0].keys():
            premium_values = [f[feature_name] for f in premium_features]
            normal_values = [f[feature_name] for f in normal_features]

            # T 检验
            t_stat, p_value = stats.ttest_ind(premium_values, normal_values)

            if p_value < 0.01:  # 差异显著
                rules[feature_name] = {
                    'premium_mean': np.mean(premium_values),
                    'premium_std': np.std(premium_values),
                    'premium_range': [
                        np.percentile(premium_values, 25),
                        np.percentile(premium_values, 75)
                    ],
                    'optimal': np.median(premium_values),
                    'significance': p_value,
                }

        return rules
```

class ContinuousLearner:
    """持续学习系统"""

    def daily_learning(self):
        """每日学习"""

        # 1. 收集昨天的反馈
        feedbacks = self.get_recent_feedbacks(days=1)

        # 2. 分析调整趋势
        insights = self.analyze_feedbacks(feedbacks)

        # 3. 更新规则
        for insight in insights:
            if insight['adjustment_rate'] > 0.3:
                # 超过 30% 用户调整，需要优化
                self.adjust_rule(
                    dimension=insight['dimension'],
                    adjustment=insight['suggested_adjustment'],
                )

        # 4. 验证更新
        self.validate_updates()

        # 5. 部署新规则
        if self.validation_passed:
            self.deploy_new_rules()
